* OOCSS
CSS의 여러 요소들을 모듈로 만들어 조합하도록 하는 방법론

    구조에 해당하는 CSS 요소
    width, height, padding, margin 등 레이아웃에 영향을 미치는 속성

    skin에 해당하는 요소: font 관련, color, background, border 들 레이아웃에 영향 미치지 않지만 시각적인 영향을 주는 속성

    요소의 스타일이 부모요소에 종속되지 않도록 해야함.
    즉, 컨텐츠의 스타일 선택자에 컨테이너의 클래스를 배제


* SMACSS
유연하고 모듈식 구조를 갖고있는 css 방법론

    CSS 코드를 역할에 따라 분리

        1. 베이스
            프로젝트의 표준 스타일을 정의
        2. 레이아웃
            헤더 메인 푸터 사이드바와 같은 웹 사이트에서 큰 틀 구성하는 모듈에 관한 규칙
        3. 모듈
            레이아웃 내에 배치되는 모든 요소를 의미 반복되는 요소를 주로 지정하기 때문에 ID 선택자 사용 안함
            비슷한 모듈안에서도 모양 조금씩 다를경우 서브클래스 생성
        4. 스테이트
            기존 스타일 덮어쓰거나 확장 시키는데 사용
            JS로 넣었다 뺐다 할 수 있는 스타일
        5. 테마
*BEM
Block Element Modifier
3가지 큰 구분으로 나누게 됨
ID, 요소 선택자 지양하고 class 선택자만 사용함 => 가중치 계산을 원활히 사용하기 위해
    1. Block   
     어디서나 재사용 가능한 부품을 의미
    2. Element
     블록을 구성하는 요소들로 block에 종속
    3. Modifier
     블록이나 Element의 모습이나 상태 또는 움직임을 정의할 때 사용
     단독으로 사용되지 않고 두번째 클래스 이름으로 사용
